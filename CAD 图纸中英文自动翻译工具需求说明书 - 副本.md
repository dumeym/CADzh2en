# CAD 图纸中英文自动翻译工具需求说明书（修订版）

## 1. 项目背景与目标
设计院业务出海，需要将大量可编辑的 CAD 图纸（DWG 格式）中的**中文文字**自动翻译为工程英语。本项目旨在开发一个自动化命令行/脚本工具，基于 Python 实现，输入 DWG 图纸，输出翻译后的 DWG 图纸（或双语对照版），过程中**不依赖 OCR**，仅处理可编辑的矢量图纸。

## 2. 适用范围
- **输入**：可编辑的 AutoCAD DWG 文件（R2004 及以上版本兼容）
- **输出**：翻译后的 DWG 文件（保持原始几何、图层、样式、位置不变）
- **处理对象**：图纸中所有用户可见的文字实体（包括但不限于以下 DXF 实体类型）
  - TEXT（单行文字）
  - MTEXT（多行文字）
  - DIMENSION（尺寸标注中的文字）
  - LEADER / MULTILEADER（引线标注中的文字）
  - ATTRIB / ATTDEF（属性定义与属性值）
  - **块定义（BLOCK）内的所有文字实体**（包括嵌套块）
  - **块引用（INSERT）对应的属性值**

## 3. 硬性约束（用户已明确）
| 约束项 | 要求 |
| ------ | ---- |
| 不采用 OCR | 完全忽略扫描图纸、图片、不可编辑 PDF |
| 仅处理可编辑 DWG | 必须先转换为 DXF 后处理，最终可输出 DWG |
| 编程语言 | Python 3.9+ |
| DXF 解析库 | 必须为 Python 原生库（推荐 `ezdxf`） |
| 术语对照表格式 | **CSV**（UTF-8 编码，两列：`中文,英文`） |
| 翻译方案 | 最便宜、最容易接入：<br>1. 优先使用用户提供的中英文术语对照表（CSV）进行精确匹配<br>2. 未匹配的普通文本调用免费/低成本的通用翻译 API（如百度翻译免费额度、Google Translate 免费版、DeepL 免费 API 等） |
| 输出格式 | 可选：① 完全覆盖原文字（英文版）② 保留原文并添加译文（双语版） |
| 文字样式 | 为保证翻译后英文的可读性（特别是长度接近原文），工具必须**自动创建一个专用的文字样式**并应用于所有翻译后的文字实体（详见 4.4） |

## 4. 功能需求

### 4.1 预处理：DWG → DXF
- 调用外部命令行转换工具（如 `libreDWG`、`TeighaConverter` 或 `ODA File Converter`）将 DWG 转换为 DXF ASCII 格式（R12 或 R2010 兼容）
- 转换后保持文件名一致，仅后缀变更
- 若输入已是 DXF，则跳过此步

### 4.2 文本提取与清洗
- 使用 `ezdxf` 递归遍历模型空间（`modelspace`）、所有图纸空间（`paperspace`）以及**所有块定义（block definitions）**
- 提取实体的以下属性：
  - 内容字符串（`text` / `plain_text`）
  - 实体句柄（`handle`）
  - 插入点坐标（`insert` 或 `dxf.insert`）
  - 图层名称
  - 文字样式、高度、旋转角度
- 过滤空格、纯数字、纯符号（例如 `%%C` 等直径符号可保留但无需翻译）
- 去除重复实体（基于句柄去重）
- **对块内文字的特殊处理**：
  - 遍历 `doc.blocks` 中每个块定义，提取其中的所有文字实体（TEXT、MTEXT、ATTRIB 等）
  - 块引用（INSERT）本身不翻译，但块定义修改后所有引用自动生效
  - 需要保留块定义的原始结构（包括嵌套块）

### 4.3 翻译引擎（可插拔）
#### 4.3.1 术语表精确匹配（第一优先级）
- 用户提供术语对照表，格式固定为 **CSV**（UTF-8 编码），两列：`中文,英文`
- 工具启动时加载术语表，构建字典（最长匹配优先，避免短词误替换）
- 扫描提取出的每段文字，对其中出现的术语进行原位替换（支持词组识别，如“给水管道”匹配“给水”+“管道”的优先级）

#### 4.3.2 通用翻译 API（第二优先级）
- 对未命中术语表的剩余字符串（按句子拆分后）调用翻译 API
- 支持多线程批量翻译以提升性能
- 建议使用 **百度通用翻译 API（免费版）**：每月免费额度 5 万字符，接入简单，支持中译英
  - 备选：Google Translate (可通过 `googletrans` 非官方库，但稳定性较差)
- 必须实现 API 调用的重试机制（最多 3 次）和错误日志

#### 4.3.3 保留不需要翻译的内容
- 尺寸数值（如 `100.5`）、直径符号、公差、特殊格式控制符（如 `\P` 表示换行）保持原样

### 4.4 文字样式处理（新增）
**目的**：中英文长度差异较大，为避免翻译后英文文字过长导致叠盖或超出边界，需要创建一个专用的文字样式来优化英文显示效果。

**具体要求**：
1. 在输出的 DXF/DWG 文件中，**自动新建一个文字样式**，命名规则固定为 `"TONGJI-FONT"`）。
2. 该样式的具体参数暂定如下（可后续通过配置文件调整）：
   - **字体（font）**：优先使用 `simplex.shx`）
   - **宽度因子（width factor）**：建议 `0.6 ~ 0.7`（用户可配置默认值 `0.65`），使英文字符更紧凑，接近中文字符的视觉宽度
   - **倾斜角度（oblique angle）**：保持 `0`（不倾斜）
   - **垂直比例（vertical scale）**：保持 `1.0`
   - **文字高度**：**不在此样式中固定**，而是继承原文的高度属性（即每个文字实体保留其原始高度，仅更换样式名称）
3. **应用范围**：所有被翻译的文字实体（包括 TEXT、MTEXT、ATTRIB、DIMENSION 的文字内容），在回填译文时将其 `style` 属性改为新创建的样式名称。
4. **对 MTEXT 的特殊处理**：MTEXT 可能自带局部格式覆盖，只能整体修改其样式，原有内联格式（如加粗、颜色）可能丢失。若必须保留，可跳过 MTEXT 的样式替换（仅替换文字内容），由用户后续批量调整。
5. **保留原样式**：工具应在修改前备份原始样式名，若最终用户选择“双语模式”（原文+译文），则建议仅译文部分使用新样式，原文保持原样式。

**实现步骤**：
- 打开 DXF 文档后，检查是否存在目标样式名，若不存在则创建。
- 创建样式时，根据运行平台选择合适字体：Windows 下注册 `Arial.ttf`，Linux 下注册 `simplex.shx` 或 `opensans.ttf`。
- 遍历所有待翻译的文字实体，修改其 `dxf.style` 为新样式名。

### 4.5 译文回填
- 基于原始 DXF 创建一个**副本**，所有修改在副本上进行
- 根据原始实体的句柄定位到目标实体，直接修改 `.dxf.text` 属性（对于 MTEXT 需同时修改 `.text` 和重建内容结构）
- **同时修改文字样式**（按 4.4 要求）
- **保持其他属性完整**：图层、坐标、高度、宽度因子、旋转、附着点等完全不变
- 若为尺寸标注，不破坏尺寸关联性（`ezdxf` 自动处理）
- 若译文长度变化导致排版拥挤，不调整文字位置（由用户后期手动微调，属于非功能需求之外的权衡）
- **块定义回填**：修改块定义中的文字实体后，无需单独处理 INSERT，所有引用自动更新

### 4.6 后处理：DXF → DWG
- 使用与预处理相同的转换工具将 DXF 转回 DWG 格式
- 输出 DWG 文件命名规则：`原文件名_EN.dwg` 或 `原文件名_双语.dwg`

## 5. 技术栈

| 类别 | 选型 | 理由 |
| ---- | ---- | ---- |
| 核心库 | `ezdxf` | Python 原生，纯 Python 实现，无需 AutoCAD 环境，官方文档完善 |
| 转换工具 | `ODA File Converter` (命令行版) | 免费可靠，支持批量转换，可用于 Linux/Windows |
| 翻译 API | 百度翻译 API（免费版） | 每月 5 万字符免费额度，超出后按量计费低至 49 元/百万字符；支持术语表干预（通过自定义术语） |
| 术语表格式 | CSV (UTF-8) | 严格限定为此格式，便于维护和版本管理 |
| 日志 | Python `logging` | 记录错误、未翻译的文本、API 超时 |
| 并行处理 | `concurrent.futures.ThreadPoolExecutor` | 多图纸或多段文本同时翻译，提升效率 |

## 6. 非功能需求

### 6.1 性能
- 单张典型图纸（500 个文字实体）处理时间不超过 30 秒（不含 API 延迟）
- 支持批量处理一个文件夹内的所有 DWG 文件

### 6.2 错误处理与可恢复性
- 遇到无法解析的 DXF 实体时，跳过并记录警告，不中断整体流程
- API 翻译失败时，保留原始中文在输出中，并在日志中标记 `[NOT_TRANSLATED]`
- 生成详细的运行报告（HTML 或 Markdown），列出每张图纸翻译了多少字符、失败了多少、使用的术语命中率

### 6.3 易于部署
- 依靠 `pip install -r requirements.txt` 安装所有依赖
- 提供单一启动入口脚本，例如 `cad_translator.py --input drawing.dwg --term-table terms.csv --output-dir ./output`
- 支持跨平台（Windows / Linux / macOS）

## 7. 使用示例

```bash
# 单文件翻译（纯英文输出）
python cad_translator.py -i 结构图.dwg -t 术语表.csv -o ./out -m replace

# 双语输出（英文在上 / 中文在下，使用 \P 拼接）
python cad_translator.py -i 给排水图.dwg -t 术语表.csv -o ./out -m bilingual

# 批量处理文件夹
python cad_translator.py -d ./图纸目录 -t 术语表.csv -o ./out
```
